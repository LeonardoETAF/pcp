# 🔒 Regras Críticas do Projeto

Estas regras devem ser seguidas obrigatoriamente.

---

## 1️⃣ Separação de Responsabilidade

- Dados → schema `catalogo`
- Lógica frontend → schema `catalogo_front`

Nunca misturar.

---

## 2️⃣ Não Duplicar Dados

- Não criar novas tabelas copiando dados existentes.
- Não replicar informações já existentes.

Sempre consumir do schema `catalogo`.

---

## 3️⃣ Backend é Responsável pelo Cálculo

Toda regra de negócio deve estar no backend.

Frontend apenas:

- Envia parâmetros
- Recebe resposta
- Exibe resultado

Nunca calcular preço no frontend.

---

## 4️⃣ Não Alterar Estrutura Base

Não modificar:

- Tabelas do schema `catalogo`
- Índices existentes
- Estrutura original

Qualquer alteração estrutural deve ser discutida antes.

---

## 5️⃣ Evolução Controlada

Qualquer nova funcionalidade deve:

- Respeitar arquitetura
- Não quebrar regras existentes
- Ser validada antes de expandir

---

Organização e clareza são prioridade.
