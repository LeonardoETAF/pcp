# 📦 PRD – Catálogo Digital Super Copo (Frontend)

---

## 📌 1. Visão Geral

Atualmente utilizamos planilhas em Excel para consulta de preços de produtos (preço do produto liso + personalização por faixa de quantidade + adicionais).

O objetivo deste projeto é substituir essas planilhas por um sistema web (frontend) dinâmico, moderno, rápido e fácil de usar.

O sistema deverá consumir exclusivamente os dados do schema `catalogo` e permitir consultas rápidas e seguras de preços, estoque e informações técnicas, reduzindo erros manuais e aumentando a escalabilidade do processo comercial.

---

## 🎯 2. Objetivo do Projeto

Criar um catálogo digital onde seja possível:

- Buscar produto pelo nome
- Informar quantidade desejada
- Escolher entre:
  - Liso
  - Serigrafia 1 cor
  - Serigrafia 2 cores
- Selecionar acabamento adicional (quando aplicável):
  - Borda
  - Hotstamping
- Visualizar automaticamente:
  - Preço parcelado (10x cartão)
  - Preço 1x cartão (5% desconto)
  - Preço PIX (10% desconto)
  - Faixa de quantidade aplicada
  - Estoque disponível
  - Informações técnicas do produto

---

## 🚨 3. Problema Atual

Hoje:

- Planilhas são enviadas manualmente
- O cliente precisa identificar a faixa correta
- Existe risco de erro de cálculo
- Atualizações exigem edição manual
- Processo pouco escalável
- Falta centralização e padronização

---

## 🧠 4. Regras de Negócio

### 4.1 Estrutura do Preço

Preço final é composto por:

```
Preço Base do Produto (Liso)
+ Personalização (por faixa de quantidade e número de cores)
+ Adicionais (acabamentos como Borda, Hotstamping, etc.)
```

A lógica deve:

- Identificar automaticamente a faixa correta com base na quantidade informada
- Aplicar adicional de 2 cores quando selecionado
- Aplicar adicional de acabamento quando selecionado

---

### 4.2 Descontos por Forma de Pagamento

- 10x cartão → preço base
- 1x cartão → 5% de desconto
- PIX → 10% de desconto

Essas regras devem ser implementadas dinamicamente no backend.

---

## 🧩 5. Funcionalidades Esperadas

### 5.1 Consulta Dinâmica

O usuário deve poder:

- Digitar o nome do produto
- Informar a quantidade
- Selecionar número de cores
- Selecionar acabamento adicional (se aplicável)

O sistema deve:

- Identificar automaticamente a faixa de quantidade correta
- Calcular o preço conforme regras de negócio
- Retornar:
  - Preço unitário
  - Valor total
  - Condições de pagamento
  - Faixa aplicada
  - Estoque disponível

---

### 5.2 Visualização em Tabela

O sistema também deve permitir:

- Visualização estilo tabela (similar à planilha)
- Filtros por categoria
- Busca por termo
- Navegação intuitiva

---

### 5.3 Informações Complementares

Cada produto deve permitir visualizar:

- Cores disponíveis
- Acessórios compatíveis
- Ficha técnica
- Dados de embalagem
- Informações relevantes adicionais

---

## 🏗️ 6. Estrutura Técnica

- Consumir dados exclusivamente do schema `catalogo`
- Criar novo schema `catalogo_front`
- Desenvolver funções (RPCs) específicas para frontend
- Não alterar estrutura do schema `catalogo`
- Não alterar RPCs de outros projetos
- Não duplicar dados
- Toda regra de cálculo deve estar no backend
- O frontend apenas envia parâmetros e exibe os resultados retornados pelas funções

---

## 🔒 7. Regras Arquiteturais

- Separar claramente:
  - Dados → `catalogo`
  - Lógica de frontend → `catalogo_front`
- Não criar tabelas redundantes
- Não replicar dados existentes
- Garantir escalabilidade futura
- Preparar estrutura para integrações futuras (ERP, WhatsApp, IA, etc.)

---

## 🛠️ 8. Fases do Projeto

### Fase 1 – Base

- Tela de busca por produto
- Campo para informar quantidade
- Seleção de cores
- Seleção de acabamento adicional
- Cálculo automático de preço
- Exibição das condições de pagamento

### Fase 2 – Evolução

- Consulta de estoque
- Ficha técnica detalhada
- Exibição de acessórios compatíveis
- Melhorias de usabilidade

### Fase 3 – Refinamento

- Interface moderna e responsiva
- Melhorias de performance
- Experiência otimizada para o time comercial
- Ajustes finos baseados em feedback real
