# 📌 Ordem de Execução – Projeto Catálogo Frontend

Este documento define a sequência correta de desenvolvimento.

Seguir exatamente esta ordem.

---

## 1️⃣ Restaurar o Banco de Dados

1. Criar novo projeto Supabase.
2. Executar:
   - `dump_estrutura_catalogo.sql`
   - `dump_dados_catalogo.sql`
3. Validar:
   - 12 tabelas no schema `catalogo`
   - Dados carregados corretamente
   - Índices criados sem erro

Não iniciar desenvolvimento antes disso estar validado.

---

## 2️⃣ Estudar o Schema `catalogo`

Antes de criar qualquer código:

- Entender estrutura das tabelas:
  - `itens`
  - `precos`
  - `tabelas_preco`
  - `cores`
  - `variantes_acabamento`
- Entender como funciona a faixa de quantidade
- Entender como preço é estruturado

Objetivo: compreender os dados antes de criar lógica.

---

## 3️⃣ Criar Schema `catalogo_front`

Criar novo schema:

```
catalogo_front
```

Este schema será responsável apenas pelas funções do frontend.

Não alterar o schema `catalogo`.

---

## 4️⃣ Criar Primeira RPC (Backend Primeiro)

Antes de criar frontend:

Criar uma função simples que:

- Receba:
  - produto
  - quantidade
  - número de cores
  - acabamento (opcional)
- Retorne:
  - preço base
  - preço 10x
  - preço 1x
  - preço PIX
  - faixa aplicada

Testar essa função via SQL Editor.

Somente após funcionar corretamente, iniciar frontend.

---

## 5️⃣ Iniciar Frontend

Frontend deve:

- Enviar parâmetros para a RPC
- Receber resultado
- Exibir dados

Frontend NÃO deve conter regra de cálculo.

---

## ⚠️ Importante

Nunca:

- Criar tabelas duplicadas
- Calcular preço no frontend
- Alterar schema `catalogo`
- Alterar estrutura existente sem alinhamento

---

Seguir essa ordem garante organização e evita retrabalho.
